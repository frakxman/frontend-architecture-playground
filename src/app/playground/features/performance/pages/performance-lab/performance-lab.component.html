📚 Understanding Change Detection

🔄

Default Strategy

¿Qué es? Angular verifica todos los componentes cuando cualquier evento ocurre (click, timer, XHR). Es fácil pero puede ser ineficiente. 

// Cada evento dispara:
@Component({{ '{' }}
  changeDetection: ChangeDetectionStrategy.Default
{{ '}' }})
  • ✅ Fácil de entender
  • ❌ Puede causar renders innecesarios
  • ❌ Menos performance en listas grandes
Valor: {{ defaultValue }} Renders: {{ defaultRenders }}
⚡

OnPush Strategy

¿Qué es? Angular solo verifica el componente cuando: @Input() cambia (nueva referencia), eventos del componente, o observables emiten. 

// Solo verifica cuando:
@Component({{ '{' }}
  changeDetection: ChangeDetectionStrategy.OnPush
{{ '}' }})
// @Input() cambia o eventos internos
  • ✅ Mucho más eficiente
  • ✅ Menos renders innecesarios
  • ❌ Requiere inmutabilidad
Valor: {{ onPushValue }} Renders: {{ onPushRenders }}

💡 Nota: El botón "Incrementar" no actualiza la vista porque muta el objeto. "Actualizar con nueva referencia" sí funciona.

🚀

Zoneless (Experimental)

¿Qué es? Angular 18+ permite ejecutarse sin Zone.js. Tú controlas manualmente cuándo verificar cambios con ChangeDetectorRef. 

// app.config.ts
provideExperimentalZonelessChangeDetection()

// Componente
constructor(private cdr: ChangeDetectorRef) {{ '{' }}

update() {{ '{' }}
  // Angular no detecta automáticamente
  this.data = nuevoValor;
  this.cdr.markForCheck();
{{ '}' }}
  • ✅ Bundle size -25%
  • ✅ Máximo control
  • ✅ Mejor performance
  • ❌ Más código manual
Valor: {{ zonelessValue }}

OnPush Strategy: Renderiza SOLO el item que cambió (si se usa trackBy y inmutabilidad).
Diferencia: {{ performanceGain }}% menos renders

🔄 TrackBy: Evita renders innecesarios

Sin trackBy: Angular destruye y recrea TODOS los items del DOM cuando la lista cambia.
Con trackBy: Angular identifica cada item por su ID y solo actualiza los que cambiaron.
